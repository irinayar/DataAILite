# DataAILite Analysis Capabilities

DataAILite is the in-memory version of DataAI. It is designed for users who want to import, view, filter, chart, summarize, and interpret data during a web session without relying on permanent application database storage.

The application is built with ASP.NET Web Forms and VB.NET. Its main analysis model is session-based: data is loaded into `DataTable` objects, stored in the user's ASP.NET session, analyzed in memory, displayed through web pages, and optionally sent to the AI interpretation workflow.

No imported data or analysis data is stored after the user logs off.

## What Makes DataAILite Different

DataAILite is lighter than the full DataAI system because it focuses on temporary, session-based analysis instead of long-term database-backed report administration.

Key characteristics:

- Data is processed in memory during the active session.
- Data does not need to be stored permanently after logout.
- No imported data or analysis results are retained after logoff.
- Users can work with imported data quickly.
- Analysis can be performed directly on ASP.NET `DataTable` objects.
- Results can be filtered, exported, charted, mapped, summarized, and interpreted.
- Optional SQLite support can be used where the Lite configuration requires a small local operational store.

## Data Sources That Can Be Analyzed

DataAILite can be programmed to analyze:

- Imported CSV and TXT files.
- Imported Excel files when import support is enabled.
- XML and JSON files converted into tabular data.
- Data returned from temporary SQLite-backed workflows.
- Existing report output stored in the current session.
- Chart, map, correlation, statistics, and matrix-balancing output generated by the application.

The most natural format for DataAILite is a rectangular table: rows, columns, column names, and values that can be loaded into a `DataTable`.

## Current In-Memory Analysis Workflows

### 1. Table Viewing and Filtering

DataAILite can load data into a session `DataTable`, display it in a grid, and preserve both the original table and the filtered table.

Possible analysis:

- View all records in the imported or generated table.
- Filter rows without changing the original data.
- Count returned records after filtering.
- Reset filters back to the original dataset.
- Export the current filtered table or report result.

### 2. Overall Statistics

DataAILite can calculate general statistics for in-memory tables.

Possible analysis:

- Record counts.
- Column counts.
- Numeric field summaries.
- Minimum and maximum values.
- Average values.
- Standard deviation.
- Missing or blank values.
- Basic data distribution notes.

This is useful for quick inspection immediately after a user imports a dataset.

### 3. Exploratory Data Analysis

DataAILite can be extended with an exploratory analysis page for any table in session.

Possible analysis:

- Identify numeric, date, and text columns.
- Show distinct value counts.
- Show most common values.
- Detect empty columns.
- Detect columns with mostly blank values.
- Detect likely ID fields.
- Detect likely category fields.
- Detect likely measure fields.
- Generate a short readable summary of the dataset.

### 4. Chart Analytics

DataAILite can analyze chart-ready data and summarize what the chart represents.

Possible analysis:

- Prepare chart data from imported CSV data.
- Summarize values by chart category.
- Identify largest and smallest values.
- Rank categories.
- Calculate totals and percentages.
- Generate chart-ready tables for web visualization.
- Export the chart data or send it to the AI interpretation workflow.

Useful chart types include bar, column, line, area, pie, scatter, histogram, and combo-style charts.

### 5. Map Analytics

DataAILite can analyze map-ready data when location columns are available.

Possible analysis:

- Identify map records with latitude and longitude.
- Count mapped locations.
- Find highest and lowest values by location.
- Summarize values by place, region, or category.
- Prepare data for map charting or Google Maps-style output.
- Generate AI-readable map summaries.

This works best for datasets with coordinates, place names, region fields, or location-based metrics.

### 6. Correlation Analysis

DataAILite can analyze relationships between numeric fields in the current in-memory dataset.

Possible analysis:

- Detect numeric columns.
- Calculate correlation coefficients between numeric fields.
- Rank strongest positive and negative relationships.
- Filter weak correlations.
- Prepare correlation tables for display or export.
- Send correlation output to AI for plain-language interpretation.

This is useful for discovering relationships inside imported CSV, spreadsheet, or report data.

### 7. Cross-Tab and Grouped Summaries

DataAILite can be programmed to build grouped summaries without saving permanent report definitions.

Possible analysis:

- Group rows by one category field.
- Group rows by multiple category fields.
- Calculate sums, counts, averages, minimums, and maximums by group.
- Build cross-tab tables from two category fields.
- Calculate percentages by row, column, or total.
- Rank groups by selected measures.

This is one of the best additions for the Lite version because it gives users useful analysis immediately after import.

### 8. Time-Based Analysis

When a dataset includes date fields, DataAILite can be programmed to summarize values over time.

Possible analysis:

- Group records by day, week, month, quarter, or year.
- Calculate period totals.
- Calculate period averages.
- Compare one period with another.
- Calculate percentage change.
- Calculate rolling totals.
- Calculate moving averages.

This supports trend-style analysis using direct ASP.NET/VB.NET calculations.

### 9. Data Quality Analysis

Because DataAILite works well with imported files, data quality checks are especially useful.

Possible analysis:

- Count blank values by column.
- Detect duplicate rows.
- Detect duplicate IDs.
- Find invalid numeric values.
- Find invalid date values.
- Flag out-of-range values.
- Flag inconsistent categories, such as spelling variations.
- Identify columns with mixed data types.
- Create a downloadable data quality report.

This would help users understand whether imported data is ready for reporting.

### 10. Matrix and Balancing Analysis

DataAILite can display and interpret matrix-balancing outputs in memory.

Possible analysis:

- Show starting matrix, target matrix, balanced matrix, and balancing coefficients.
- Compare differences between starting, target, and balanced values.
- Calculate maximum cell differences.
- Summarize row and column totals.
- Explain balancing results in plain language.
- Export matrix results.

This is useful for allocation, reconciliation, survey weighting, proportional adjustment, and similar business workflows.

### 11. AI-Assisted Explanation

DataAILite can send the current table, filtered data, chart analysis, map analysis, correlation output, statistics, or matrix-balancing result to the AI page.

Possible analysis:

- Explain what the imported data shows.
- Summarize key patterns.
- Explain chart results.
- Explain map results.
- Explain correlations.
- Explain statistics.
- Explain matrix-balancing results.
- Answer follow-up questions about the current session data.

The AI workflow should be used for explanation, summarization, and plain-language interpretation. The numeric calculations should remain in ASP.NET/VB.NET so results are consistent and repeatable.

## ASP.NET Features That Can Be Programmed for DataAILite

Good extensions for this project include:

- A dedicated "Analyze Imported Data" page.
- A data profiling panel for the current session table.
- A data quality report page.
- A grouped summary builder with dropdowns for category and value fields.
- A cross-tab builder with row, column, and value selectors.
- A time summary builder for date fields.
- A top/bottom ranking report.
- A percentage-of-total report.
- A compare-two-groups report.
- A compare-two-files report.
- A correlation threshold filter.
- A chart recommendation helper based on selected fields.
- A map readiness checker for latitude/longitude data.
- A matrix result viewer and exporter.
- One-click "Explain with AI" buttons for each analysis result.
- Export for generated analysis tables and reports.

Depending on the report type and configured output workflow, results can be exported to CSV, Excel, PDF, and Word formats.

## Best Fit

DataAILite is best suited for quick, temporary, in-memory business data analysis:

- Import a file.
- Inspect the data.
- Filter records.
- Generate statistics.
- Build charts.
- Build grouped summaries.
- Check data quality.
- Review correlations.
- Analyze locations.
- Analyze matrix outputs.
- Export results.
- Ask AI to explain the results.

Because DataAILite is in-memory and session-based, it is especially useful for lightweight analysis, demonstrations, training, temporary datasets, and users who do not want to configure a full reporting database.

## Important Limits

DataAILite analysis depends on the size of the imported data, available server memory, ASP.NET session limits, configured import limits, and timeout settings. Very large datasets are better handled by the full DataAI database-backed workflow.

Since the Lite version is designed around temporary data, features that require long-term history, permanent scheduling, large shared dashboards, or enterprise report administration fit better in the full DataAI application.
